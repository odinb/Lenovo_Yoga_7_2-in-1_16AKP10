## Prerequisites:
- Manjaro KDE installed, kernel 6.12+
- calf and lsp-plugins-lv2 installed: `sudo pacman -S calf lsp-plugins-lv2`

This procedure assumes the 6.18.12-kernel, adjust as necessary.

## Step 1: Patch the HDA Codec Module
The ALC287 codec on this laptop (subsystem ID 17aa:391d) is not yet recognized by the stock kernel, so volume control won't work without a patched module.

1a. Get the kernel source matching your running kernel:<br />
`uname -r` # e.g. 6.18.12-1-MANJARO<br />
`cd ~`<br />
`sudo pacman -S linux618-headers` # required when compiling software that interacts with the kernel<br />
```
git clone --depth=1 --branch v6.18.12 \
https://github.com/torvalds/linux.git ~/linux-source`
````
`tar -xvf linux-6.18.12.tar.xz`

OR manually download just the realtek codec files from the kernel mirror

1b. Apply the patch:<br />
Edit `~/linux-6.18.12/sound/hda/codecs/realtek/alc269.c` and find the `alc269_fixup_tbl[]` array. Add this line in the 17aa: section (before the terminating entry):<br />
`HDA_CODEC_QUIRK(0x17aa, 0x391d, "Lenovo Yoga 7 2-in-1 16AKP10", ALC287_FIXUP_YOGA9_14IAP7_BASS_SPK_PIN),`<br />
Personally I put it on the line right after the Lenovo "14AKP10" model.

1c. Build the module:<br />
`sudo pacman -S base-devel`
```
make -C /usr/lib/modules/$(uname -r)/build \
  M=~/linux-6.18.12/sound/hda/codecs/realtek \
  modules
```

1d. Install the patched module:<br />
Compress:<br />
`zstd ~/linux-6.18.12/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko`

Backup original:<br />
```
sudo cp /usr/lib/modules/$(uname -r)/kernel/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko.zst \
  /usr/lib/modules/$(uname -r)/kernel/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko.zst.bak
```

Install patch:<br />
```
sudo cp ~/linux-source/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko.zst \
  /usr/lib/modules/$(uname -r)/kernel/sound/hda/codecs/realtek/
```
`sudo depmod -a`<br />
`sudo reboot`

1e. Verify after reboot:<br />
`sudo dmesg | grep -i "ALC287\|391d\|fixup"` # Should show: "ALC287: picked fixup for codec SSID 17aa:391d"

## Step 2: Fix ALSA Speaker Levels
By default the hardware speaker volume is set to 64% which sounds weak. Set to 100% and save:<br />
`amixer -c 1 sset 'Speaker' 100%`<br />
`amixer -c 1 sset 'Bass Speaker' on`<br />
`amixer -c 1 sset 'Master' 100%`<br />
`sudo alsactl store`

Note: Speaker is set to 100%, to avoid hardware clipping distortion when EQ bass boost is applied, this might need to be reduced.

## Step 3: PipeWire EQ Filter Chain
This adds a permanent equalizer in PipeWire — no extra apps needed, works on every boot.

3a. Create the config:<br />
```
# Create PipeWire config directory
mkdir -p ~/.config/pipewire/pipewire.conf.d

# Create EQ configuration
cat > ~/.config/pipewire/pipewire.conf.d/eq-laptop.conf << 'EOF'
context.modules = [
  { name = libpipewire-module-filter-chain
    args = {
      node.description = "Laptop Speaker EQ"
      media.name = "Laptop Speaker EQ"
      filter.graph = {
        nodes = [
          { type=builtin name=eq1  label=bq_lowshelf  control={ "Freq"=60.0   "Gain"=3.5 } }
          { type=builtin name=eq2  label=bq_peaking   control={ "Freq"=80.0   "Q"=0.7 "Gain"=3.5 } }
          { type=builtin name=eq3  label=bq_peaking   control={ "Freq"=120.0  "Q"=1.0 "Gain"=4.5 } }
          { type=builtin name=eq4  label=bq_peaking   control={ "Freq"=250.0  "Q"=1.0 "Gain"=2.5 } }
          { type=builtin name=eq5  label=bq_peaking   control={ "Freq"=400.0  "Q"=1.0 "Gain"=3.5 } }
          { type=builtin name=eq6  label=bq_peaking   control={ "Freq"=900.0  "Q"=2.0 "Gain"=-2.5 } }
          { type=builtin name=eq7  label=bq_peaking   control={ "Freq"=2000.0 "Q"=1.5 "Gain"=2.5 } }
          { type=builtin name=eq8  label=bq_peaking   control={ "Freq"=2800.0 "Q"=2.0 "Gain"=1.5 } }
          { type=builtin name=eq9  label=bq_peaking   control={ "Freq"=4000.0 "Q"=1.5 "Gain"=-2.0 } }
          { type=builtin name=eq10 label=bq_peaking   control={ "Freq"=6000.0 "Q"=1.2 "Gain"=-3.5 } }
          { type=builtin name=eq11 label=bq_highshelf control={ "Freq"=8000.0 "Gain"=-3.0 } }
        ]
        links = [
          { output="eq1:Out"  input="eq2:In" }
          { output="eq2:Out"  input="eq3:In" }
          { output="eq3:Out"  input="eq4:In" }
          { output="eq4:Out"  input="eq5:In" }

          { output="eq7:Out"  input="eq8:In" }
          { output="eq8:Out"  input="eq9:In" }

      }
      capture.props = {
        node.name = "effect_input.eq_speaker"
        media.class = "Audio/Sink"
        audio.channels = 2
        audio.position = [ FL FR ]
      }

        node.name = "effect_output.eq_speaker"
        node.target = "alsa_output.pci-0000_04_00.6.HiFi__Speaker__sink"
        audio.channels = 2
        audio.position = [ FL FR ]
      }
    }
  }
  { name = libpipewire-module-filter-chain
    args = {
      node.description = "Headphones EQ"
      media.name = "Headphones EQ"
      filter.graph = {
        nodes = [
          { type=builtin name=eq1  label=bq_lowshelf  control={ "Freq"=105.0  "Gain"=4.0 } }
          { type=builtin name=eq2  label=bq_peaking   control={ "Freq"=380.0  "Q"=0.7 "Gain"=-3.0 } }
          { type=builtin name=eq3  label=bq_peaking   control={ "Freq"=1000.0 "Q"=1.5 "Gain"=-1.5 } }
          { type=builtin name=eq4  label=bq_peaking   control={ "Freq"=2500.0 "Q"=1.2 "Gain"=2.5 } }
          { type=builtin name=eq5  label=bq_peaking   control={ "Freq"=4500.0 "Q"=1.5 "Gain"=3.0 } }
          { type=builtin name=eq6  label=bq_peaking   control={ "Freq"=6000.0 "Q"=2.0 "Gain"=-1.5 } }
          { type=builtin name=eq7  label=bq_highshelf control={ "Freq"=9000.0 "Gain"=-2.0 } }
        ]
        links = [
          { output="eq1:Out" input="eq2:In" }
          { output="eq2:Out" input="eq3:In" }
          { output="eq3:Out" input="eq4:In" }
          { output="eq4:Out" input="eq5:In" }
          { output="eq5:Out" input="eq6:In" }
          { output="eq6:Out" input="eq7:In" }

      capture.props = {
        node.name = "effect_input.eq_headphones"
```

3d. Now the WirePlumber EQ-auto-switch script:
```
# Create WirePlumber EQ-auto-switch directory
mkdir -p ~/.config/wireplumber/scripts/

# Create WirePlumber EQ-auto-switch configuration
cat > ~/.config/wireplumber/scripts/auto-switch-eq.lua << 'EOF'
log = Log.open_topic("s-auto-switch-eq")

local headphone_sink = "alsa_output.pci-0000_04_00.6.HiFi__Headphones__sink"

local om_meta = ObjectManager {
  Interest { type = "metadata",
    Constraint { "metadata.name", "=", "default" }
  }
}

local om_hp = ObjectManager {
  Interest { type = "node",
    Constraint { "node.name", "=", headphone_sink }
  }
}

local function set_default_sink(name)
  local metadata = om_meta:lookup {
    Constraint { "metadata.name", "=", "default" }
  }
  if metadata then
    log:info("Switching default sink to: " .. name)
    metadata:set(0, "default.configured.audio.sink", "Spa:String:JSON",
      Json.Object { ["name"] = name }:to_string())
  end
end

om_hp:connect("object-added", function(_, node)
  log:info("Headphones plugged in")
  set_default_sink("effect_input.eq_headphones")
end)

om_hp:connect("object-removed", function(_, node)
  log:info("Headphones unplugged")
  set_default_sink("effect_input.eq_speaker")
end)

om_meta:activate()
om_hp:activate()
EOF
```
3e. Now the Login autostart script — sets default sink, headphone EQ if 3.5mm plugged in, otherwise speaker EQ:
```
# Set default sink
cat > ~/.local/bin/set-default-sink.sh << 'EOF'
#!/bin/bash
sleep 3
# Check if headphones sink exists (headphones plugged in)
HP_SINK=$(wpctl status | grep "effect_input.eq_headphones" | awk '{print $1}' | tr -d '.')
SP_SINK=$(wpctl status | grep "effect_input.eq_speaker" | awk '{print $1}' | tr -d '.')

if [ -n "$HP_SINK" ] && wpctl status | grep -q "HiFi__Headphones__sink"; then
    wpctl set-default "$HP_SINK"
else
    wpctl set-default "$SP_SINK"
fi
EOF
chmod +x ~/.local/bin/set-default-sink.sh
```
To activate, it will be needed to logout user, and then log back in.

3f. Now the KDE autostart entry
```
# KDE autostart entry
cat > ~/.config/autostart/set-default-sink.desktop << 'EOF'

Type=Application
Name=Set Default Audio Sink (EQ)
Exec=/home/brodersen/.local/bin/set-default-sink.sh
Terminal=false

```

## Troubleshooting
Volume control stops working:<br />
`wpctl set-volume @DEFAULT_AUDIO_SINK@ 0.5`<br />
`wpctl set-mute @DEFAULT_AUDIO_SINK@ 0`

EQ not applying (audio sounds flat):
Check EQ sink exists:<br />
`pactl list sinks short`<br />
Set it as default (use the ID shown for effect_input.eq):<br />
`wpctl set-default <ID>`<br />
Restart Brave/Firefox to reconnect to new default sink

Speaker level reset to 64%:<br />
`amixer -c 1 sset 'Speaker' 100%`<br />
`sudo alsactl store`

## Kernel update 
After kernel update — patch will be lost, rebuild and reinstall snd-hda-codec-alc269.ko.zst against new kernel headers. Volume control will stop working as the symptom.

Re-patching After Kernel Update:<br />
1. Check new kernel version:<br />
`uname -r # e.g. 6.18.15-1-MANJARO`

2. Install new kernel headers:<br />
`sudo pacman -S linux618-headers` # Replace 618 with your kernel series number (e.g. 619, 620)

3. Verify the patch is still in your source:<br />
`grep "391d" ~/linux-6.18.12/sound/hda/codecs/realtek/alc269.c`<br />
Should show: HDA_CODEC_QUIRK(0x17aa, 0x391d, "Lenovo Yoga 7 2-in-1 16AKP10", ALC287_FIXUP_YOGA9_14IAP7_BASS_SPK_PIN),<br />
If the line is there, skip step 4.

4. If source is gone or needs updating, re-apply the patch:<br />
Find the fixup table in alc269.c and add the line back.<br />
`vi ~/linux-6.18.12/sound/hda/codecs/realtek/alc269.c`<br />
Find: SND_PCI_QUIRK(0x17aa, 0x38b5, "Legion Slim 7 16IRH8"<br />
and: HDA_CODEC_QUIRK(0x17aa, 0x391c, "Lenovo Yoga 7 2-in-1 14AKP10"<br />
Add after it: HDA_CODEC_QUIRK(0x17aa, 0x391d, "Lenovo Yoga 7 2-in-1 16AKP10", ALC287_FIXUP_YOGA9_14IAP7_BASS_SPK_PIN),

5. Clean old build artifacts<br />
`cd ~/linux-6.18.12/sound/hda/codecs/realtek/`
```
make -C /usr/lib/modules/$(uname -r)/build \
  M=~/linux-6.18.12/sound/hda/codecs/realtek \
  clean
```

6. Rebuild against new kernel headers
```
make -C /usr/lib/modules/$(uname -r)/build \
  M=~/linux-6.18.12/sound/hda/codecs/realtek \
  modules
```

7. Compress, backup and install:<br />
Compress<br />
`zstd ~/linux-6.18.12/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko`

Backup new stock module:
```
sudo cp /usr/lib/modules/$(uname -r)/kernel/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko.zst \
  /usr/lib/modules/$(uname -r)/kernel/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko.zst.bak
```

Install patched module
```
sudo cp ~/linux-6.18.12/sound/hda/codecs/realtek/snd-hda-codec-alc269.ko.zst \
  /usr/lib/modules/$(uname -r)/kernel/sound/hda/codecs/realtek/
```
`sudo depmod -a`<br />
`sudo reboot`

8. Verify after reboot<br />
`sudo dmesg | grep -i "ALC287\|391d\|fixup"`<br />
Must show: "ALC287: picked fixup for codec SSID 17aa:391d"<br />
If it shows 38b5 instead, the patch didn't take — check step 6

Note: If Manjaro updates to a new kernel series (e.g. 6.19), you'll need to install linux619-headers and the new kernel source may need the patch re-applied to a fresh copy of alc269.c. The patch line itself won't change — only the kernel version you build against changes.
Tip: Once this patch lands officially in the upstream kernel ([Bug 221210](https://bugzilla.kernel.org/show_bug.cgi?id=221210)), none of this will be needed anymore — the fix will be included automatically in every kernel update.

A few ways to track it:<br />
1. Watch the kernel bugzilla directly:
[Bug 221210](https://bugzilla.kernel.org/show_bug.cgi?id=221210)<br />
Click "Add me to CC list" at the bottom — you'll get email when there's any activity, comments, or status changes.

2. Watch the upstream kernel git for the fix.<br />
Once you think it might be merged, search the kernel changelog:<br />
`curl -s https://raw.githubusercontent.com/torvalds/linux/master/sound/hda/codecs/realtek/alc269.c | grep "391d"`<br />
When that returns your line, it's in mainline.

3. Check when Manjaro picks it up.<br />
After a kernel update, just check dmesg - if it still works without your patched module:<br />
`sudo dmesg | grep "391d"`<br />
If it shows up WITHOUT you having installed the patched module, it landed!

4. Simplest ongoing check after any kernel update.<br />
Before repatching, test if it's still needed:<br />
`sudo dmesg | grep -i "fixup"`<br />
If it shows 17aa:391d naturally → patch is in kernel, you're done with this patching.<br />
If it shows 17aa:38b5 → you still need to repatch.
